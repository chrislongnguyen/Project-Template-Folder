# LEARNING HIERARCHY
## Facts → Information → Wisdom → Expertise
### Derived From: Wiki — Effective Learning

---

## The 4 Levels of Learning

### Level 1: KNOWLEDGE (Facts & Information)
| Question | Purpose |
|----------|---------|
| **So What?** | Why is this important? (Relevance) |
| **What is it?** | Definition and scope |
| **What else?** | Related concepts, alternatives |
| **How does it work?** | Mechanisms, processes |

### Level 2: UNDERSTANDING (Why)
| Question | Purpose |
|----------|---------|
| **Why does it work?** | Underlying principles |
| **Why not?** | When and why does it fail? |

### Level 3: WISDOM (Application)
| Question | Purpose |
|----------|---------|
| **So What?** | How can we benefit from this? |
| **Now What?** | What should we do next? |

### Level 4: EXPERTISE (Mastery)
| Question | Purpose |
|----------|---------|
| **What is it NOT?** | Common misconceptions |
| **How does it NOT work?** | Anti-patterns to avoid |
| **What If?** | Alternatives if this fails |
| **Now What?** | How can we do better than others? |

---

## Application
Use this hierarchy when filling out research templates in `2-PLAN/learning/research/`.
The minimum acceptable depth for project research is Level 2 (Understanding).
Level 3-4 is required for core architectural decisions.

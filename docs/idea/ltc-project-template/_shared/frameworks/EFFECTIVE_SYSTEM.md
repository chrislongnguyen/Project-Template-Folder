# THE 7-COMPONENT EFFECTIVE SYSTEM MODEL
### Derived From: Section 3 — Effective System Diagnosis & Design

---

## Core Concept
Every system (tool, process, project, organization) has 7 components. When something underperforms, diagnose ALL 7 before attempting to fix. Most failures are misdiagnosed because people fix the wrong component.

---

## The 7 Components

| # | Component | Abbreviation | Key Question | Example |
|---|-----------|-------------|--------------|---------|
| 1 | Effective User | **EU** | Is the right person/role doing this? | Wrong skill set assigned to task |
| 2 | Effective Actions | **EA** | Are the right actions being taken? | Doing the wrong work |
| 3 | Effective Outcomes | **EO** | Are the right outcomes defined? | Success criteria missing or wrong |
| 4 | Effective Principles | **EP** | Are the governing principles sound? | Bad design principles |
| 5 | Effective Operating Environment | **EOE** | Is the environment supportive? | Wrong infrastructure, missing tools |
| 6 | Effective Operating Tools | **EOT** | Are the tools adequate? | Inadequate technology stack |
| 7 | Effective Operating Procedure | **EOP** | Is the procedure complete? | Missing or broken SOP |

## How to Use for Diagnosis
When a system is underperforming:
1. Check each component systematically (do NOT jump to conclusions)
2. Identify which component(s) are the actual root cause
3. Apply UBS/UDS analysis to the failing component
4. Fix the root cause, not the symptom

## How to Use for Design
When designing a new system:
1. Define all 7 components explicitly
2. Validate each against the 3 Pillars (Sustainability, Efficiency, Scalability)
3. Identify the UBS for each component
4. Document in `2-PLAN/architecture/SYSTEM_DESIGN.md`

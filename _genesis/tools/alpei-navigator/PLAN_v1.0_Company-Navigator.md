## 2. Skill Connection Chains

### Chain 1: Learning Pipeline (L Row)
```
learn (L-D coordinator)
  -> learn-input (L-D, L-S) — scopes what to learn
    -> learn-research (L-B) — deep research execution
      -> learn-review (L-V) — review & approve research
        -> learn-spec (L-V -> A-B bridge) — extract specifications
          -> learn-structure (L-B) — create doc structures
          -> learn-visualize (L-B) — create visuals
```

### Chain 2: Process Pipeline (A -> P -> E Rows)
```
ltc-brainstorming (A-D) — ideation with trade-offs
  -> ltc-writing-plans (P-D) — write implementation plans
    -> ltc-execution-planner (P-B, P-S) — task breakdown
      -> ltc-task-executor (E-B) — multi-agent execution
```

### Chain 3: Quality Loop (E -> I Rows)
```
team-architect (E-D) — design agent teams
  -> [build phase] (E-B)
    -> team-reviewer (E-V) — code & design review
      -> feedback (I-B) — capture issues as GitHub Issues
        -> [feeds back to A-D next iteration]
```

### Chain 4: Compliance (Crosses All V Cells)
```
ltc-brand-identity + ltc-naming-rules + ltc-rules-compliance
  -> enforced at ALL V gates
  -> ltc-skill-creator (meta: creating new skills must pass compliance)
```

### Chain 5: Session Lifecycle (Crosses All Rows)
```
setup -> session-start -> [work with any skills] -> compress
  -> [new session] -> resume -> [continue work] -> session-end
  -> session-end syncs with ltc-clickup-planner / ltc-notion-planner
```

### Chain 6: Research (Supports L and A Rows)
```
deep-research (L-B, A-B) — deep research with citations
root-cause-tracing (I-B, I-D) — causal chain analysis
  -> both feed into decision-making at any D cell
```

### Chain 7: Core Workflow Engine
```
align-specialist (A-D) — entry point for alignment
dsbv skill — triggers at every D->S->B->V transition across ALL rows
```

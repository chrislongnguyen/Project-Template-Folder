# research

> "What external knowledge has been captured and is ready for synthesis?"

## Purpose

Research captures from external sources — books, articles, courses, conversations.

External knowledge that is captured but never linked to project work adds noise without value. This folder holds processed research notes — not raw captures (those belong in `inbox/`). A note enters `research/` only after it has been read, summarized, and connected to at least one project or workstream.

## What This Contains

| Content Type | Description |
|-------------|-------------|
| Book/article notes | Processed summaries with key insights and source citation |
| Course captures | Structured notes from training, workshops, and courses |
| Conversation notes | Synthesized takeaways from expert interviews or significant discussions |

## How It Connects

```
vault/inbox/ (raw captures)
    │
    └──> vault/research/ (processed external knowledge)
              │
              ├──> 2-LEARN/ — research outputs are formal LEARN workstream input
              ├──> vault/projects/ — research linked to projects that apply it
              └──> _genesis/frameworks/ — significant insights may surface as framework updates
```

## Pre-Flight Checklist

- [ ] Confirm each research note cites its source (author, title, date, URL)
- [ ] Verify each note links to at least one project or workstream that applies its insights
- [ ] No orphaned or stale artifacts

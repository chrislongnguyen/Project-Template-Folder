## Research: Hermes Agent Evaluation

### What It Is

Open-source (MIT) autonomous agent runtime by Nous Research, released Feb 2026. v0.6.0. 23.8K GitHub stars, 1,447 open issues.

**Core bet:** persistent 5-layer memory + autonomous skill creation (SKILL.md every 15 tasks) + RL training loop (Atropos).

### GAN Verdict: DO NOT ADOPT (score: -2.5/10)

| Dimension | Wt | Net Score | Notes |
|-----------|-----|-----------|-------|
| Solves #1 pain (orchestration) | 25% | -7 | Single-agent-first; our gap is multi-agent coordination |
| Memory / context continuity | 20% | +2 | Real innovation but buggy (3 open critical memory issues) |
| Skill self-improvement | 15% | +5 | Genuinely novel, nothing like it in our stack |
| Security / compliance | 15% | -7 | Sandbox bypass #4146 + silent failures |
| Integration cost | 10% | -5 | Must wrap in context-packaging, hooks, DSBV format |
| Maturity / support | 10% | -5 | Pre-1.0, no SLA, research org backing |
| Cost / local inference | 5% | +3 | Nice-to-have |

### Key Findings

**Strengths:**
- 5-layer memory (conversation → Theory of Mind → skills → metadata → env state)
- Auto-writes SKILL.md after complex tasks, refines every 15 uses
- MCP-native, broad provider support (Ollama, vLLM, llama.cpp)
- 10+ messaging integrations (Telegram, Discord, Slack, Signal)

**Disqualifiers:**
- Memory bugs in critical path: silent write failures (#2771), double flush (#3059), message loss (#4469)
- Sandbox bypass OPEN and CRITICAL (#4146) — `execute_code` bypasses terminal approval
- Multi-agent added 6 weeks ago (v0.6.0) — parent→child only, no peer coordination
- No equivalent to our 4-tier enforcement (hooks → rules → skills → agents)
- DSBV incompatible — would need wrapper code negating "just works" benefit

**What our stack already does better:**
- 4 MECE agents with context packaging + hook validation
- Deterministic enforcement via PreToolUse hooks
- ALPEI 5×4 matrix with binary acceptance criteria
- QMD + tiered memory system (functional if clunky)

### One Idea Worth Stealing

Hermes' autonomous skill creation *pattern* — not the framework. Proposed: post-Validate hook that detects novel tool chains and drafts SKILL.md candidates for human review. ~20-line hook, zero framework dependency.

### Watch List

- Revisit at v1.0 (est. Q3 2026) — memory bugs likely fixed, multi-agent may mature
- If Nous ships declarative orchestration + peer agents, becomes compelling

# LEARN / IDM — Insights & Decision-Making Learning

Research and knowledge artifacts for the Insights and Decision-Making subsystem.

## Cascade Position

IDM is the terminal stage of the LEARN pipeline.

- Upstream: DA — inherits analytical outputs and validated findings
- Downstream: none within LEARN; IDM outputs feed stakeholder-facing deliverables

IDM learning identifies which decision frameworks, reporting patterns, and stakeholder communication approaches translate DA findings into actionable decisions.

## Contents

| Artifact Type | Naming Pattern | Purpose |
|---|---|---|
| Decision Framework Research | `idm-framework-{name}.md` | Evaluation of decision-making frameworks |
| Reporting Patterns | `idm-reporting-patterns.md` | Reference patterns for insight reporting |
| Stakeholder Communication Specs | `idm-stakeholder-{role}.md` | Communication requirements per stakeholder role |
| Escalation Research | `idm-escalation.md` | Frameworks for decision escalation paths |

## Routing

Outputs from this directory feed `1-ALIGN/` (stakeholder communication standards) and `4-EXECUTE/docs/` (reporting templates).

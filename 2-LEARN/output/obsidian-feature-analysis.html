Graceful degradation
Confirmed		 If Obsidian CLI fails (app not running, vault not registered), agent falls back to grep. Floor = current behavior. No broken sessions. Not a testing crutch. Implementation Priority — RICE Score

Why RICE? Reach × Impact × Confidence ÷ Effort. Confidence naturally down-weights speculative features (V1 flywheel, V7 layer bleed, V9 stubs) without manual bias. Effort up-ranks one-line fixes over multi-day builds. Three constraints override score: (1) P0 unblocking actions run first regardless of score, (2) dependency ordering enforced within phases, (3) deferred items score 0 by definition.

Scale: Reach 0–3 (never → every session) · Impact 1–5 (marginal → decisive/S2-level) · Confidence 0–1 (speculative → A/B confirmed) · Effort story points (1 = <1hr · 2 = half-day · 3 = full day · 5 = multi-day · 8 = week+)

P0 — Unblocking Prerequisite (runs before all scores apply)

SKILL.md syntax rewrite — All CLI features use wrong syntax (--flags instead of key=value). Until fixed, L1/L2/L5/L6/L8/L9/L10/L12/L13 are all non-functional regardless of RICE score. Remove graph:walk (L3). Fix AP4 to opt-out (L8). Add search:context (L5). Add mandatory grep sweep (L9).
Effort: ~2hrs · Do this first, before anything else.

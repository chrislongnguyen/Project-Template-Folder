Vinh's Whiteboard — AI OS Desired Outcomes

Transcribed from Vinh's whiteboard session. This is the primary "why" for making Human Adoption (N8) the top UBS. Correct & Safe is assumed; the gaps are Efficient and especially Scalable. Main cause identified: AI agent memory is text (.md) files — efficient, but not scalable.
Desired Outcome Correct & Safe Efficient Scalable Vinh's Diagnosis
Human–human alignment ? ✓ ✗ Alignment exists session-to-session but breaks across team members without shared memory infrastructure
Learn ? ? ✗ Agents learn within sessions; knowledge doesn't compound across sessions or team members at scale
Human PM – AI Agent ? ? ✗ Direction-setting works for 1 human + 1 agent; breaks when multiple agents or members are involved
↳ Plan ? ✓ ✗✗ Planning artifacts exist but are per-session; no shared plan state visible to all agents simultaneously
↳ Execute ? ✓ ✗✗ Single-agent execution is efficient; multi-agent coordination at scale requires cmux + shared workspace
Improvement ? ✓ ✗ Individual retros work; IMPROVE workstream outputs don't feed back into next iteration reliably
Vinh's Solution Architecture
Root Cause
AI Agent's memory = Text (.md) files
Efficient for 1 human + 1 agent.
Not scalable for 8 members + N agents.
Main scalability bottleneck = memory architecture.
Solution Stack
NotebookLM → LEARN → Human ← REVIEW
Captured & Organized Knowledge
SAVE → Linked .md files (Obsidian convention)
QUERY → AI Agents
Memory System: CLAUDE.md + MEMORY.md + Routing + .md collection
Obsidian CLI bridging role: (1) Memory connection via OBSIDIAN SKILL · (2) "Too much effort" = adoption blocker — must reduce friction. N8 Human Adoption Training Package directly addresses barrier #2.

Session Compression Optimization
New		 3 4 1.0 2 6.0 NEW-P1 Refactor /compress to write frontmatter-heavy session files (~500 tokens) instead of narrative summaries (~5K tokens). Auto-update MEMORY.md Briefing Card date+state. /resume costs 35K tokens (15–20% of 200K Sonnet window). Briefing Cards go stale (5/6 projects frozen at March 17). Session Lifecycle Deprecation
New		 3 2 1.0 1 6.0 NEW-P1 Deprecate /session-start and /session-end. User hasn't used them "for quite a while." /resume + /compress + Stop hook cover the full workflow. Dead code creates confusion. New team members don't know which skills to use. Dead code paths waste tokens when skills are loaded. Fewer choices = less confusion. Clear directive: use /compress and /resume only. N7
Open Items Carry-Forward
New		 3 3 0.8 2 3.6 NEW-P1 /compress extracts unresolved items from session. /resume surfaces them as "Open from last session." Unchecked checkboxes propagate across sessions. Open items die between sessions. "I had an open question about X" — lost. Must re-derive from git log. Session boundaries are artificial — work-in-progress doesn't end when a session ends. Sees "Open items from last session" at resume. Decides to continue or close. Extracts open_items from /compress frontmatter, surfaces at /resume. ArtemXTech: "Nothing is done until you verify it." Steve Phelps: backlog items tagged #backlog in daily notes, carried forward across days. Add open_items field to session file frontmatter. /resume reads and surfaces. Depends on N1 (compress must write rich frontmatter first).
N2
2-Pass Resume
New		 3 4 0.8 2 4.8 NEW-P1 /resume reads frontmatter of last 3 sessions first (~500 tokens), then body of most relevant 1 (~2K tokens). Total: ~3–5K tokens instead of 35K. Every session burns 35K tokens on recall. On 200K Sonnet, only 62% of context window remains for actual work. Source: ArtemXTech dashboard pattern. Says /resume — gets faster, cheaper recall. No behavior change needed. 2-pass: scan frontmatter → selective body load → surface open_items (carry-forward). ArtemXTech reads Obsidian Base table views (frontmatter only), not full files: "Context-efficient because you only feed the table view." 10x cheaper context. Refactor /resume skill. Depends on N1 (compress must write rich frontmatter first). Depends on N7 for open_items surfacing.

Multi-agent at scale requires isolation + observability. Tabs don't provide either. Named workspaces = named context. Create cmux quickstart guide (part of N8 training package). Add to training slide deck. Long needs fluency before team rollout. Source: ArtemXTech.

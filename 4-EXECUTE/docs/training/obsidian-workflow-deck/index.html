Scene 2 — Vault Structure

Your Vault Layout

LT Capital Partners
Folder Map
📁 vault-root/ 1-ALIGN/ charter, decisions, OKRs 2-LEARN/ research, specs, learning pipeline 3-PLAN/ risks, drivers, roadmap 4-EXECUTE/ src, tests, config, docs 5-IMPROVE/ retros, metrics, changelog DAILY-NOTES/ standup log, daily context MISC-TASKS/ ad-hoc work not tied to workstream PEOPLE/ stakeholder notes PLACES/ location / venue context TEMP-IN/ inbound reference material TEMP-OUT/ export staging area THINGS/ tools, resources, objects AI-AGENT-MEMORY/ agent persistent context inbox/ ⬅ agents write HERE only 0-REUSABLE-RESOURCES/ templates/ bases/
Two Zones
ALPEI Zone (5 folders)
Workstream artifacts live here. Each folder maps 1:1 to a DSBV phase. You create and curate content here using Templater. Bases dashboards read from these folders.
Operational Zone (8 folders + inbox/)
Daily context, people, tasks, and staging. No formal DSBV structure required — just capture what matters. Bases include DAILY-NOTES views.
Agent Write Rule — Critical
Agents (Claude Code, ltc-builder) write only to inbox/ and AI-AGENT-MEMORY/. Review inbox/ before moving files to canonical folders. This prevents silent file corruption.
Setup
Run once on fresh clone: ./scripts/setup-obsidian.sh Creates all folders, copies templates + bases. Idempotent — safe to re-run.

How Bases work
Bases read frontmatter fields from your vault files in real time. Change status: draft → in-progress in any file and it updates instantly across all dashboards. No manual data entry. No sync lag.
Open a dashboard
In Obsidian: navigate to 0-REUSABLE-RESOURCES/bases/ → open any .base file. Click the Bases icon in the ribbon for quick access.

What PM Does (Obsidian)
  • 1
    Write the task in MISC-TASKS/ or 3-PLAN/
    Include: EO (what success looks like), inputs, constraints
  • 2
    Open Cursor in the repo folder
    Cursor reads the same Git-backed vault — no file transfer needed
  • 3
    Run /dsbv build or ltc-builder
    Agent reads DESIGN.md + SEQUENCE.md → produces artifact → writes to inbox/
  • 4
    Review inbox/ in Obsidian
    Accept: move to canonical folder. Reject: delete + note feedback.

Non-negotiable handoff rule
Never manually edit files in 4-EXECUTE/src/ or 4-EXECUTE/tests/. Even small edits can break agent context. Capture your intent in PLAN or MISC-TASKS and let the agent implement it.
Scene 7 — Quick Reference

Recap + Reference Card

LT Capital Partners
Status Vocabulary (S2)
draft in-progress in-review validated archived
  • draft — created, not started
  • in-progress — actively being built
  • in-review — awaiting human approval
  • validated — human-approved (you set this)
  • archived — deprecated, kept for record
Plugin Stack
  • Bases — live dashboard views (14 dashboards)
  • Templater — auto-fill frontmatter on new notes
  • Daily Notes — core plugin, daily standup entry
  • Dataview — query engine backing some views
  • Kanban — task board view for MISC-TASKS
Agent Write Rules
  • Agents write to: inbox/ + AI-AGENT-MEMORY/ only
  • You move: accepted files from inbox/ to canonical folders
  • Never manually edit: 4-EXECUTE/src/, tests/, scripts/
  • Sync: Git only — no Obsidian Sync required
7-Scene Summary
1 Workspace Model: Obsidian (thinking) + Cursor (building). Git-backed sync.
2 Vault: 5 ALPEI + 8 operational + inbox/. Run setup-obsidian.sh once.
3 Daily: Open DAILY-NOTES → standup → check Bases → review inbox/.
4 Create: Templater fills 9-field frontmatter automatically in <1 min.
5 Dashboards: 14 Bases in 0-REUSABLE-RESOURCES/bases/. Auto-refresh.
6 Handoff: Write task in PLAN → run ltc-builder → review inbox/ → approve.
7 Status vocabulary: draft → in-progress → in-review → validated → archived. All lowercase in frontmatter.
